**Автор проекта:** DenBOND  
**Версия:** 1.0  
**Платформы:** Desktop + Mobile (полноценная адаптация)

---

## 📌 1. Общее описание

Разработать **полноценную онлайн-игру в шахматы** для двух игроков с реалтайм-синхронизацией через комнаты. Проект должен идеально работать как на ПК, так и на **мобильных устройствах** (iOS Safari, Android Chrome). Готов к публикации на GitHub и развёртыванию на Render.com / Railway.app с получением публичной ссылки для игры.

**Концепция:** Игрок создаёт комнату → получает 6-значный код → делится ссылкой с другом → второй подключается → играют в шахматы с чатом, таймером и всеми плюшками.

**Стиль:** Современный, тёмный, минималистичный, красивый.  
**Обязательная подпись в футере:** `Made with ❤️ by DenBOND`

## 🎮 4. Функциональные требования

### 4.1. Главное меню
- Крупный заголовок **"♟️ BondChess"**
- Кнопка **"Создать игру"** → генерирует 6-значный код (буквы + цифры, uppercase)
- Поле ввода **"Код комнаты"** + кнопка **"Войти"**
- Выбор **режима сложности подсказок** перед стартом:
  - **Лёгкий** — подсказки доступны сразу
  - **Нормальный** — подсказки появляются через 30 секунд после начала хода игрока
  - **Сложный** — подсказки отключены
- После создания — кнопка **"📋 Скопировать ссылку"** (формат: `site.com/?room=ABC123`)
- Поддержка входа по прямой ссылке `?room=XXXXXX` (автоподключение)
- Красивое отображение ошибок (не alert!)

### 4.2. Игровой экран

**Верхняя инфо-панель:**
- Код комнаты + кнопка копирования
- Твой цвет (⚪ Белые / ⚫ Чёрные)
- Чей ход сейчас
- Статус соперника (онлайн/отключён)

**Доска:**
- Адаптивный размер: на ПК до 560px, на мобилке — почти вся ширина экрана
- **Квадратное соотношение** через `aspect-ratio: 1`
- Координаты по краям (a-h, 1-8)
- Для чёрных — **доска перевёрнута** (свои фигуры внизу)
- Цвета: светлые `#f0d9b5`, тёмные `#b58863`
- **Подсветки:**
  - Выбранная клетка (жёлтая)
  - Возможные ходы (точка в центре)
  - Возможное взятие (красная рамка)
  - Последний ход (подсветка "откуда/куда")
  - Шах королю (красный фон клетки короля)
- Поведение подсветок зависит от выбранной сложности:
  - **Лёгкий:** при выборе фигуры показывать возможные ходы и типы ходов (обычный/взятие) сразу
  - **Нормальный:** аналогичные подсказки показывать только после 30 секунд бездействия в текущем ходу
  - **Сложный:** не показывать подсказки возможных ходов и взятий (оставить только служебные подсветки: последний ход, шах)

**Таймеры:**
- 10 минут каждому (в константе, легко менять)
- Отсчёт только в ход игрока
- Подсветка активного (зелёный)
- При < 30 сек — красный + мигание
- При 0 — победа соперника

**Боковая панель:**
- **История ходов** — алгебраическая нотация, моноширинный шрифт, автоскролл
- **Чат** — ввод + Enter для отправки, цвет отправителя (⚪/⚫), автоскролл, защита от XSS, макс. 200 символов

**Кнопки:**
- Сдаться (с подтверждением)
- Предложить ничью (модалка у соперника)
- Новая игра (после партии)
- Выйти в меню

### 4.3. Логика игры
- **Все правила шахмат:** обычные ходы, рокировка, взятие на проходе, **превращение пешки**
- **Превращение пешки:** модальное окно с выбором Ферзь/Ладья/Слон/Конь
- **Валидация:** на клиенте (UX) + обязательно на сервере (безопасность)
- Режим сложности влияет только на визуальные подсказки, но **не влияет на правила игры и валидацию ходов**
- **Конец игры:**
  - Мат → победа
  - Пат → ничья
  - 50 ходов без прогресса → ничья
  - Троекратное повторение → ничья
  - Недостаточный материал → ничья
  - Сдача / истечение времени → победа соперника
- **Модалка окончания** с результатом и кнопкой "Новая игра"

### 4.4. Сервер
- Хранилище комнат в памяти (объект `{ roomId: {...} }`)
- События: `createRoom`, `joinRoom`, `move`, `chatMessage`, `resign`, `offerDraw`, `acceptDraw`, `declineDraw`, `disconnect`
- В `createRoom` сохранять выбранный режим сложности (`easy | normal | hard`) как параметр комнаты
- Автоудаление комнаты при уходе игрока
- **Защита от читинга:** сервер ВСЕГДА проверяет ход через `chess.js`
- Создатель комнаты = белые, второй = чёрные
- Логирование событий в консоль

### 4.5. Контракт Socket.IO

**Client → Server:**
- `createRoom ({ difficulty })` — создать с выбранной сложностью (`easy | normal | hard`)
- `joinRoom (roomId)` — войти
- `move ({ roomId, from, to, promotion })` — ход
- `chatMessage ({ roomId, message })` — сообщение
- `resign ({ roomId })` — сдаться
- `offerDraw ({ roomId })` — предложить ничью
- `acceptDraw ({ roomId })` / `declineDraw ({ roomId })`

**Server → Client:**
- `roomCreated ({ roomId, color, difficulty })`
- `roomJoined ({ roomId, color, fen, difficulty })`
- `gameStart ({ fen })`
- `moveMade ({ fen, move, turn, history })`
- `timerUpdate ({ white, black })`
- `chatMessage ({ color, message })`
- `drawOffered`
- `gameOver ({ result, reason })`
- `opponentLeft`
- `errorMsg (message)`
- `invalidMove`

## 📱 5. МОБИЛЬНАЯ ВЕРСИЯ (ОБЯЗАТЕЛЬНО, критично важно!)

### 5.1. Адаптивная вёрстка
- **Mobile-first подход** — сначала мобилка, потом ПК
- Breakpoints:
  - `< 600px` — мобильный (доска сверху, панель снизу вкладками)
  - `600-900px` — планшет (компактный вид)
  - `> 900px` — десктоп (всё рядом)
- Использование `flexbox` и `grid`
- `viewport` meta-тег: `width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no` — запрет зума

### 5.2. Touch-управление
- **Ход по тапу:** тап на фигуру → подсветка возможных ходов → тап на клетку назначения
- Альтернатива — drag & drop пальцем (если реально реализовать чисто, иначе только тапы)
- Увеличенные зоны нажатия (минимум 40x40px на клетку)
- Отключить `user-select` на доске (`user-select: none`)
- Отключить выделение текста и контекстное меню при долгом тапе: `-webkit-touch-callout: none`
- Отключить подсветку нажатия: `-webkit-tap-highlight-color: transparent`
- Haptic feedback при ходе (через `navigator.vibrate(30)`)

### 5.3. Мобильный layout
- **Инфо-панель сверху:** компактная, в одну строку, иконки вместо длинных текстов
- **Таймеры:** сверху и снизу доски (у соперника сверху, у себя снизу)
- **Доска:** занимает ~95% ширины экрана
- **Под доской — табы:**
  - 📜 История
  - 💬 Чат
  - ⚙️ Управление (сдаться/ничья/выйти)
- Только одна вкладка видна одновременно — экономим место
- В чате: большой инпут, не перекрывается клавиатурой

### 5.4. Клавиатура на мобилках
- При открытии клавиатуры в чате — layout не должен ломаться
- Использовать `window.visualViewport` API для корректировки
- Кнопка "Отправить" рядом с инпутом (большая, удобная для пальца)
- `input` с атрибутом `enterkeyhint="send"` и `autocomplete="off"`

### 5.5. Размеры шрифтов и элементов
- Фигуры на мобилке: `font-size: clamp(28px, 9vw, 50px)`
- Кнопки: минимум `44x44px` (Apple HIG)
- Отступы между кликабельными элементами: минимум 8px
- Весь текст читаем без зума (мин. 14px)

### 5.6. Ориентация
- Поддержать и **портретную**, и **ландшафтную**
- В ландшафтной — доска слева, панель справа (как на десктопе, но компактнее)

### 5.7. PWA (опционально, но желательно)
- `manifest.json` — чтобы игру можно было "установить на главный экран"
- Иконка приложения (можно Unicode ♟️ как эмодзи-иконку через SVG)
- Полноэкранный режим (`display: standalone`)
- Тема `theme-color` под цвет фона

### 5.8. Производительность на мобилках
- Никаких тяжёлых библиотек
- Анимации через `transform` и `opacity` (GPU-ускорение)
- Не перерисовывать всю доску при каждом событии — только изменённые клетки (если возможно)
- Debounce для resize-событий

## 🚀 7. Требования к коду

- **Чистый, читаемый** с комментариями в ключевых местах
- `const` / `let`, стрелочные функции, async/await где нужно
- Обработка ошибок (try/catch на сервере)
- **Никаких `alert()` и `confirm()`** — только кастомные модалки/тосты
- Все настройки (время, порт, лимиты) — **в константах сверху файла**
- Разделение на логические функции (не один мегаскрипт)
- Защита от XSS в чате (экранирование HTML)
- Валидация входных данных на сервере
